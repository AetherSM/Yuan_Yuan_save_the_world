# 校园跑腿与交易系统 - 指定流程图 (Mermaid)

本文件按您的要求，仅保留 4 类流程图：
1. 管理员模块设计流程图
2. 跑腿员接单流程图
3. 用户购买商品流程图
4. 用户发布任务流程图

---

## 1. 管理员模块设计流程图

```mermaid
flowchart TB
    A0[管理员登录] --> A1[进入管理后台]
    A1 --> A2{选择管理模块}

    A2 --> U1[用户管理]
    U1 --> U2[按手机号/昵称/角色查询]
    U2 --> U3[禁用或启用账号]
    U2 --> U4[调整用户角色]

    A2 --> P1[商品管理]
    P1 --> P2[查询全平台商品]
    P2 --> P3[审核待审核商品]
    P2 --> P4[强制修改商品状态]

    A2 --> E1[跑腿任务管理]
    E1 --> E2[查询全平台跑腿任务]
    E2 --> E3[审核通过 status:0->1]
    E2 --> E4[审核拒绝 status:0->6]
    E2 --> E5[强制改状态 status:任意->指定]

    A2 --> O1[订单管理]
    O1 --> O2[查询商品订单]
    O2 --> O3[强制改状态 status:1/2/3/4/5/6/7]
```

---

## 2. 跑腿员接单流程图

```mermaid
flowchart LR
    R0[跑腿员进入任务大厅] --> R1[查看待接单任务 status:1]
    R1 --> R2[点击接单]
    R2 --> R3{并发更新是否成功}
    R3 -- 否 --> R4[提示: 接单失败/已被领取]
    R3 -- 是 --> R5[订单状态 status:1->2]
    R5 --> R6[跑腿员开始配送 status:2->3]
    R6 --> R7[用户确认送达]
    R7 --> R8[订单完成 status:3->4]
    R8 --> R9[系统结算: 用户扣款, 跑腿员入账]
```

---

## 3. 用户购买商品流程图

```mermaid
flowchart TB
    B0[用户浏览商品] --> B1[加入购物车]
    B1 --> B2[提交订单]
    B2 --> B3{库存校验/商品状态校验}
    B3 -- 失败 --> B4[提示库存不足或商品不可售]
    B3 -- 通过 --> B5[创建订单 status:1 待支付]
    B5 --> B6[用户支付]
    B6 --> B7[状态变更 status:1->2 待发货]
    B7 --> B8[商家发货]
    B8 --> B9[状态变更 status:2->3 待收货]
    B9 --> B10[用户确认收货]
    B10 --> B11[状态变更 status:3->4 已完成]
    B11 --> B12[系统给商家记收入流水]
```

---

## 4. 用户发布任务流程图

```mermaid
flowchart TB
    C0[用户填写任务信息] --> C1[提交跑腿任务]
    C1 --> C2[系统创建订单 status:0 待审核]
    C2 --> C3[管理员审核]
    C3 --> C4{审核结果}
    C4 -- 通过 --> C5[任务上架 status:0->1 待接单]
    C4 -- 拒绝 --> C6[任务结束 status:0->6 审核拒绝]
    C5 --> C7[跑腿员接单 status:1->2]
    C7 --> C8[配送中 status:2->3]
    C8 --> C9[用户确认完成 status:3->4]
```

---

## 5. 状态码速查

### 商品订单状态
- `1`: 待支付
- `2`: 待发货
- `3`: 待收货
- `4`: 已完成
- `5`: 已取消
- `6`: 退款中
- `7`: 已退款

### 跑腿订单状态
- `0`: 待审核
- `1`: 待接单
- `2`: 已接单
- `3`: 配送中
- `4`: 已完成
- `5`: 已取消
- `6`: 审核拒绝
- `7`: 退款中
- `8`: 已退款
